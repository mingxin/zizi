# 产品需求文档 (PRD) - zizi AI识字伴侣 (Demo版)

| 文档版本 | V0.2 (Revised) |
| :--- | :--- |
| **项目名称** | **zizi AI识字伴侣 (zizi Literacy AI)** |
| **面向用户** | 3-6岁儿童及其家长 |
| **产品形态** | 移动端 Web App / 微信小程序 (轻量级验证) |
| **核心场景** | 孩子拍摄物品，AI识别并关联汉字，讲述趣味短故事 |

---

## 1. 项目背景与目标
### 1.1 背景
3岁儿童处于图像认知向符号认知过渡的敏感期。传统识字卡片枯燥，缺乏互动。利用大模型的多模态能力，可以将现实世界与汉字学习连接起来，实现“万物皆可学”。

### 1.2 Demo 核心目标
*   **验证核心体验：** 确认“拍照 -> 识图 -> 讲故事”的延迟是否在儿童可接受范围内（<3秒为佳，Demo允许5秒内）。
*   **验证吸引力：** 观察孩子听到AI生成的个性化语音后的反应。
*   **技术跑通：** 跑通 视觉识别 + LLM生成 + TTS语音合成 的API串联流程。

---

## 2. 用户角色 (Persona)
*   **一级用户（操作者/体验者）：** 3岁儿童。
    *   *特征：* 不识字，依赖听觉和视觉，手指精细动作差，注意力短，喜欢重复。
*   **二级用户（辅助者）：** 家长。
    *   *特征：* 负责打开应用，辅助拍照，关注内容是否安全、教育是否准确。

---

## 3. 功能范围 (Scope)

| 模块 | 包含 (In Scope) | 不包含 (Out of Scope) |
| :--- | :--- | :--- |
| **核心交互** | 拍照/上传图片、语音播放、汉字展示 | 连续对话、语音打断、纠音评分 |
| **内容范围** | 仅限约 20-50 个基础独体字 (如：日、月、口、手、猫、果等) | 复杂的词组、成语、书写练习 |
| **账户体系** | 无需登录，即开即用 | 用户注册、学习进度记录、付费系统 |
| **视觉表现** | 简单的汉字演变动图 (GIF) 或静态图 | 复杂的3D互动、Live2D虚拟人 |

---

## 4. 详细功能需求

### 4.1 首页 (Home)
*   **UI布局：** 极简设计。屏幕中央一个巨大的、带呼吸动效的“相机/眼睛”按钮。
*   **引导语音：** 自动播放：“嗨！我是**zizi**。快点一下大眼睛，带我去看看你的玩具吧！”
*   **交互：** 点击按钮 -> 唤起系统相机。

### 4.2 识别与处理页 (Loading/Processing)
*   *痛点解决：* AI处理需要时间（约3-5秒），孩子会不耐烦。
*   **UI逻辑：** 屏幕显示“**zizi**正在吃下这张照片...”的可爱动画（例如一个小精灵在大口嚼东西）。
*   **后台逻辑 (核心 API 链条)：**
    1.  **图像上传：** 压缩图片上传至服务器。
    2.  **图像识别 (Vision API)：** 识别主体（例如：一只橘猫）。
    3.  **汉字映射 (Logic)：**
        *   系统判断：识别物体是否在【目标字库】中？
        *   *Case A (命中)：* 识别到猫 -> 匹配字“猫”。
        *   *Case B (未命中)：* 识别到电视遥控器 -> 强制关联相关字 -> 匹配字“手”（因为是用手拿的）或“电”。
    4.  **内容生成 (LLM API)：** 生成一段约50字的童趣文案。
    5.  **语音合成 (TTS API)：** 将文案转为高表现力的童声。

### 4.3 结果互动页 (Result)
这是产品的核心体验界面。

*   **UI布局：**
    *   **上部：** 刚才拍摄的照片（加上发光的边框）。
    *   **中部：** 巨大的目标汉字（例如“猫”）。
        *   *特效：* 如果资源允许，展示“猫实物图 -> 甲骨文 -> 楷书”的3帧渐变。
    *   **下部：** 播放按钮（默认自动播放）。
*   **音频内容示例：**
    *   “哇！我看到了一只可爱的小老虎！哦不对，是**猫**！这只猫眯着眼睛在睡觉呢。你看这个‘猫’字，左边是它的爪子，右边像不像它卷起来的尾巴？你好呀，小猫！”
*   **操作：**
    *   按钮：“再玩一次”（返回首页）。
    *   按钮：“这是什么字？”（重播读音）。

## 5. AI Prompt (提示词) 策略需求
*开发者需在后台配置并固化以下 Prompt 逻辑：*

**Role (角色设定):**
> 你是“**zizi**”，一个3岁的汉字小精灵。你说话声音软萌，喜欢用拟声词（哇、咦、嘻嘻）。你的任务是通过图片教孩子认识一个简单的独体字。

**Constraint (约束):**
1.  **字库限制：** 只能从 [日, 月, 水, 火, 口, 目, 耳, 手, 足, 山, 石, 田, 禾, 猫, 狗, 鸟, 鱼] 列表中选一个最相关的字。
2.  **关联逻辑：** 如果图片里没有上述物体，请发挥想象力关联。例如拍“沙发”，可以说“在这个软软的沙发上，我们最喜欢用‘耳’朵听故事啦”，从而引出“耳”字。
3.  **字数：** 50字以内，口语化，不要说教。

**Input (输入):** [图片]
**Output Format (输出格式 JSON):**
```json
{
  "target_char": "猫",
  "story_text": "哇！一只大黄猫！它正瞪着圆圆的眼睛看我呢...",
  "mood": "excited" // 用于控制TTS的情绪
}
```

---

## 6. 非功能性需求 (NFR)
1.  **响应速度：** 从拍照结束 to 声音开始播放，需控制在 **5秒** 以内（可使用流式传输优化）。
2.  **容错性：** 无论拍什么（哪怕是一片黑），AI都不能报错，必须兜底回复：“哎呀太黑了zizi看不清，但我猜你在找‘黑’色的东西对不对？”
3.  **安全性：** 上传的图片不进行云端永久存储，处理完即删除（保护家庭隐私）。

## 7. 开发路线图 (Roadmap for Demo)

本阶段不设具体时间表，按以下**依赖顺序**推进开发：

### 阶段一：技术验证 (PoC - Proof of Concept)
*   **目标：** 确保“眼睛”和“嘴巴”能连通。
*   **内容：**
    *   申请 Vision / LLM / TTS 服务的 API Key。
    *   编写一个简单的 Python 脚本：上传一张本地图片 -> 调用 Vision 识别 -> 调用 LLM 生成文本 -> 调用 TTS 生成音频文件。
    *   验证生成的故事质量和语音的情感是否符合“zizi”的人设。

### 阶段二：前端原型构建 (Frontend)
*   **目标：** 让孩子能拿在手上操作。
*   **内容：**
    *   搭建 App 或小程序框架 (Uni-app/Flutter)。
    *   实现核心功能：调用手机摄像头拍照、展示 Loading 动画、播放网络音频。
    *   UI 还原：完成首页大按钮和结果页的简单布局。

### 阶段三：后端逻辑与Prompt调试 (Backend & Logic)
*   **目标：** 让 zizi 变聪明。
*   **内容：**
    *   搭建后端服务器接口，接收前端上传的图片。
    *   **重点工作：** 调试 Prompt (提示词)。测试各种复杂场景（如光线暗、物体模糊、背景杂乱），确保 zizi 都能关联到字库中的汉字，且逻辑不生硬。
    *   实现字库映射逻辑（确保不跑题）。

### 阶段四：联调与体验测试 (Integration & Testing)
*   **目标：** 像一个真正的产品。
*   **内容：**
    *   前后端联调，优化响应速度（考虑流式输出）。
    *   寻找真实用户（3-5岁儿童）进行测试。
    *   观察孩子是否能听懂 zizi 的话，是否会模仿发音。
