# 产品需求文档 (PRD) - zizi AI识字伴侣

| 文档版本 | V1.0 (Current) |
| :--- | :--- |
| **项目名称** | **zizi AI识字伴侣 (zizi Literacy AI)** |
| **面向用户** | 3-6岁儿童及其家长 |
| **产品形态** | 移动端 Web App (Next.js + FastAPI) |
| **核心场景** | 孩子拍摄物品，AI识别并关联汉字，讲述趣味短故事 |

---

## 1. 项目背景与目标

### 1.1 背景
3岁儿童处于图像认知向符号认知过渡的敏感期。传统识字卡片枯燥，缺乏互动。利用大模型的多模态能力，可以将现实世界与汉字学习连接起来，实现"万物皆可学"。

### 1.2 产品目标
- **核心体验**："拍照 -> 识图 -> 讲故事"全流程延迟控制在5秒内
- **教育价值**：通过AI将实物与汉字关联，增强学习趣味性
- **技术目标**：跑通 视觉识别 + LLM生成 + TTS语音合成 的完整链路

---

## 4. 详细功能需求

### 4.1 首页 (Home)

**UI布局：**
- 极简设计，屏幕中央一个巨大的、带呼吸动效的相机按钮（200x200px圆形）
- 顶部显示 "zizi" 品牌名和 "AI识字伴侣" 副标题
- 右上角设置按钮（齿轮图标）

**交互：**
- 点击相机按钮 -> 唤起系统相机（优先后置摄像头）
- 点击设置按钮 -> 进入设置页

**引导文案：**
> "点一下大眼睛，带zizi去看看你的玩具吧！"

---

### 4.2 识别与处理页 (Processing)

**UI布局：**
- 顶部返回按钮（左上角）
- 中央显示 "zizi正在吃掉这张照片..." 带浮动动画
- 显示拍摄照片的圆形缩略图，应用咀嚼动画
- 底部3个跳动圆点表示加载中

**后台逻辑 (核心 API 链条)：**
1. **图像上传**：压缩图片上传至服务器（限制10MB）
2. **图像识别 (Vision API)**：调用阿里云 qwen-vl-max 识别图片主体
3. **汉字映射**：
   - 从选定的字库中选择最相关的汉字
   - 优先匹配字库中的字，如未命中则强制关联
4. **内容生成 (LLM)**：生成50字以内童趣文案
5. **语音合成 (TTS)**：异步生成故事音频和单字音频

**错误处理：**
- 任何错误显示友好提示："哎呀，zizi看不清这个东西，让我们再试一次吧！"
- 2秒后自动返回首页

---

### 4.3 结果互动页 (Result)

**UI布局：**
- **顶部**：返回按钮（左上角）
- **上部**：拍摄照片（240x240px，圆角24px，发光边框）
- **中部**：巨大的目标汉字（120px字体，渐变颜色，点击可重播读音）
- **下部**：故事内容（气泡对话框样式，点击播放/停止）
- **底部**：操作按钮区

**音频内容结构：**
- 开场吸引：拟声词/感叹词（哇、咦、喵喵喵）
- 实物关联：描述拍摄内容
- 汉字讲解：简单介绍字形/字义
- 互动结尾：问候/提问

**示例：**
> "喵喵喵！一只可爱的小猫！它有长长的胡须，会抓老鼠。猫字左边是反犬旁，右边是苗。你好呀，小猫！"

**操作按钮：**
- "▶ 听zizi讲故事" / "⏹ 停止播放"：播放/停止故事音频
- "再玩一次"：返回首页，重置所有状态

## 5. AI Prompt 策略

### 5.1 角色设定
> 你是"zizi"，一个3岁的汉字小精灵。你说话声音软萌，喜欢用拟声词（哇、咦、嘻嘻）。你的任务是通过图片教孩子认识一个简单的独体字。

### 5.2 约束条件
1. **字库限制**：只能从用户选定的字库列表中选择最相关的字
2. **关联逻辑**：如果图片里没有字库中的字，发挥想象力强制关联
3. **字数**：50字以内，口语化，不要说教
4. **安全**：内容必须适合3-6岁儿童，无暴力、恐怖元素

### 5.3 输出格式
```json
{
  "char": "选中的汉字",
  "story": "50字以内的有趣小故事，童趣十足"
}
```

## 6. 技术架构

### 6.1 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| **前端** | Next.js + React + TypeScript | Next.js 16, React 19 |
| **样式** | Tailwind CSS | v4 |
| **后端** | Python + FastAPI | Python 3.9+ |
| **AI 视觉** | 阿里云 Dashscope (qwen-vl-max) | - |
| **语音合成** | 阿里云 Dashscope (qwen3-tts-flash) | - |
| **降级方案** | Mock数据 + 浏览器 Web Speech API | - |

### 6.2 API 接口

| 接口 | 方法 | 功能 | 响应 |
|------|------|------|------|
| `/api/process` | POST | 处理图片，返回汉字、故事、音频 | `{target_char, story_text, audio_url, char_audio_url}` |
| `/api/voices` | GET | 获取可用音色列表 | `{voices: [...]}` |
| `/api/word-libraries` | GET | 获取字库列表 | `{libraries: [...]}` |
| `/api/tts` | POST | 文本转语音 | `{use_browser, audio_url}` |
| `/api/tts/preview` | POST | 音色预览 | `{use_browser, audio_url}` |

### 6.3 数据流

```
用户拍照
    │
    ▼
前端上传图片 + 字库ID + 音色ID
    │
    ▼
后端接收 → Vision API识别 → LLM生成故事 → TTS合成音频
    │
    ▼
前端展示结果 ← 返回JSON ← 异步音频生成
```

## 7. 非功能性需求 (NFR)

### 7.1 性能要求
- **响应时间**：拍照到声音播放 ≤ 5秒（目标3秒）
- **图片限制**：单张图片 ≤ 10MB
- **超时处理**：TTS生成 8-10秒超时，降级到浏览器语音

### 7.2 兼容性
- **浏览器**：Chrome、Safari、微信内置浏览器
- **设备**：iOS Safari、Android Chrome
- **降级方案**：无API Key时使用Mock数据

### 7.3 安全性
- **隐私**：上传的图片不进行云端永久存储，处理完即释放
- **内容**：AI生成内容需适合儿童，过滤不当内容

### 7.4 可访问性
- **字体大小**：主要文字 ≥ 18px，汉字展示 120px
- **触摸目标**：按钮尺寸 ≥ 44x44px
- **动画**：支持 reduced-motion 偏好设置
